### 第4章 実装・ハードウェア構成

#### 4.1 μ-cavity最適化

本節ではμ-cavityの品質係数 \(Q\) を最大化しつつモード体積 \(V_{\text{mode}}\) を最小化する最適化手法について記述する。最適化では以下の目的関数
$$
J = -\alpha \ln Q(R, L, t) + \beta V_{\text{mode}}(R, L)
$$
を用いる。ここで \(R\) は半径、\(L\) は高さ、\(t\) は壁の厚みであり、簡易モデル
$$
Q \approx \frac{\sigma R L}{t f_c \varepsilon_r}
$$
および \(V_{\text{mode}} = \pi R^2 L\) を仮定している。`tools/cavity_optimize.py` は L-BFGS-B 法により \(J\) を数値的に最小化する。実行例は以下の通り。

```bash
$ python tools/cavity_optimize.py
Optimal R (m): 0.0198
Optimal L (m): 0.0500
Optimal t (m): 0.0007
Minimum J: 2.31
```

シミュレーションから得られた \(J\) と半径の関係を以下に示す。本章の計算結果は `tools/` 以下のスクリプトによって自動生成されたものである。

#### 4.2 SQUIDカップリング

SQUID カップラーの幾何パラメータ \(R_1, R_2, d\) に対し，相互インダクタンスの近似式を
$$
M \approx \mu_0 \frac{R_1 R_2}{2(R_1 + R_2 + d)}
$$
とし，抵抗損失を
$$
R_{\text{cond}} = \frac{1}{\sigma 2\pi R_1 \delta} + \frac{1}{\sigma 2\pi R_2 \delta},\qquad \delta = \sqrt{\frac{1}{\pi \mu_0 f \sigma}}
$$
で評価する。目的関数は
$$
J = \bigl(g_{\text{target}} - k_{\text{couple}} M\bigr)^2 + \lambda R_{\text{cond}}
$$
である。`tools/squid_coupler.py` を実行すると以下のような出力が得られる。

```bash
$ python tools/squid_coupler.py
Optimal R1 (m): 0.0049
Optimal R2 (m): 0.0061
Optimal d (m): 0.0023
Minimum J: 4.8e-3
```

以上より，目標結合強度を満たすパラメータ組を決定できる。

### 第5章 制御・デコヒーレンス抑制

#### 5.1 フラックスポンプ最適化

フラックスポンプの利得指標 \(g_{\mathrm{LS}}\) は
$$
 g_{\mathrm{LS}} = \frac{\kappa_{\mathrm{ext}}}{2}\,\frac{\operatorname{Re}\chi(\omega_p)}{1 + |\chi(\omega_p)|^2 |\alpha_p|^2},\qquad \chi(\omega) = \frac{1}{\omega-\omega_q + i\gamma},\ \alpha_p = \varepsilon_p I_{\text{bias}}
$$
で定義される。`tools/fluxpump_optimize.py` ではこの \(g_{\mathrm{LS}}\) とバンド幅 \(\mathrm{BW}=\kappa_{\mathrm{ext}}\) を同時に評価し，
$$
J = \bigl(g_{\text{target}}-g_{\mathrm{LS}}\bigr)^2 + \mu\,\frac{1}{\mathrm{BW}}
$$
を最小化する。実行例を示す。

```bash
$ python tools/fluxpump_optimize.py
ω_p [GHz]: 7.468
ε_p: 0.12
I_bias: 0.95
g_LS: 0.171
Minimum J: 1.2e-4
```

得られた \(g_{\mathrm{LS}}\) とポンプ周波数の関係例を以下に示す。生成されたグラフは `docs/plot/fluxpump_g_vs_freq.png` に保存される。

#### 5.2 Dynamical Decouplingシーケンス

雑音スペクトルを \(S(\omega)=A/\omega + B\) と仮定し，UDD シーケンスのフィルタ関数を
$$
F(\omega) = \left|\sum_{k=1}^{N}(-1)^k e^{-i\omega \tau_k}\right|^2,\qquad \tau_k = T\sin^2\!\left(\frac{\pi k}{2(N+1)}\right)
$$
と定める。デコヒーレンス率は
$$
\Gamma_{\text{dec}} = \int_{\omega_{\text{min}}}^{\omega_{\text{max}}}\! \frac{S(\omega) F(\omega)}{\omega^2}\, d\omega
$$
で評価され，\(T_2=1/\Gamma_{\text{dec}}\) が得られる。`tools/dd_simulation.py` の実行例を示す。

```bash
$ python tools/dd_simulation.py
Γ_dec: 3.4e-9
1/T2: 2.9e8
```

#### 6.3 QRSC／PLV／トモグラフィ検証手順

##### 📡 QRSC（Quantum Random State Characterization）

-   乱数基底にパルス適用
-   出力確率分布を多数回測定
-   乱数列と理論期待値の一致度を評価

##### 🔁 PLV（Phase-Locked Verification）

-   ロックイン検出による位相ノイズ計測
-   VNA を用いて参照信号とのロック検証
-   レーザーリファレンスとの比較で位相分散を評価

##### 🔍 トモグラフィ測定仕様（例）

| 基底 | 投影回数 | 精度 (1σ) |
|------|----------|-----------|
| X    | 1,000    | ±0.02     |
| Y    | 1,000    | ±0.02     |
| Z    | 1,000    | ±0.01     |

-   合計試行：3,000 回（1 回 ≈ 2 ms → 合計 ≈ 6 秒）
-   データ保存量：1 試行 4 B × 3,000 = 12 kB
-   10 回平均化：120 kB / 60 秒

##### ⚖️ 実験負荷の評価ポイント

-   単一 vs 平均化の S/N トレードオフ
-   時間とデータ量のバランス設計が必要

#### 6.4 冷却・真空系の要件

##### 🧊 冷却ステージとヒートロード

-   冷却段階：300 K → 77 K → 4 K → 0.1 K
-   放射熱ロード：

-   伝導熱ロード：ケーブル・支持構造を通じて伝導
-   合計ヒートロード目標：100 μW 以下（0.1 K 維持）

#### 7.1 パラメータ相互依存モデル

実験で抽出した共鳴周波数や <span class="math-inline">Q</span> 因子、ノイズ係数 <span class="math-inline">A,B</span>、デコヒーレンス率 <span class="math-inline">\\Gamma\_\{\\text\{dec\}\}</span> を理論パラメータへマッピングした。表7.1 に主要パラメータの対応関係を示す。

| 理論パラメータ        | 実験値由来      |
|-----------------------|-----------------|
| <span class="math-inline">\\omega\_0</span> (fc\_GHz)  | metrics.json    |
| <span class="math-inline">Q\_\{\\text\{loaded\}\}</span>   | metrics.json    |
| <span class="math-inline">A,B</span>                 | noise\_fit.json |
| <span class="math-inline">\\Gamma\_\{\\text\{dec\}\}</span> | t2.json         |

温度ドリフトから求めた係数を <span class="math-inline">\\alpha\_\{\\mathrm\{temp\}\}</span> とおき、
$$\\omega(t)=\\omega\_0\\bigl(1-\\alpha\_{\\mathrm{temp}} t\\bigr)

$$
と補正する。熱負荷の総和も同時にパラメータ更新へ反映した。

#### 8.2 スケーラビリティとエコシステム形成

##### 📈 拡張ステップ：8Q → 16Q → 83Q の設計課題

-   **配線密度の限界**
-   8Q：Coax ケーブル × 16本
-   16Q：32本 → 多段コネクタ対応
-   83Q：64本以上 → 光結合 + スパイラル配線を検討
-   **冷却負荷**
-   多キャビティの束ね → 0.1Kステージの熱負荷急増
-   解決策：セラミック支持体、超薄型銅ストリップケーブル
-   **キャビティ間クロストーク**
-   リスク：電磁結合による誤動作
-   対策：メタマテリアルシールド・フェライトコア挿入

#### 8.3 リスク評価とマイルストーン計画

##### 🚨 技術リスクと対策

-   **コヒーレンス時間の限界**
-   現状：T₁ ≈ 1 ms、T₂ ≈ 0.8 ms
-   リスク：多キャビティ時の熱雑音・高周波ノイズ増加

-   配線数：8Q → 16Qで 8×8 → 16×16に増加
-   リスク：最適化計算コスト／クロストーク増
-   対策：Q-NoC（再構成可能な配線網）導入

-   **製造リードタイム**
-   キャビティ製作に2ヶ月以上
-   真空系の立ち上げ・リークテスト：1ヶ月

-   各フェーズに応じた機材・作業者数を事前調整
-   並列スケジューリングでリスク最小化
